## v2.5.1 변경사항

### 포스트 글쓰기 목록도 4단 통일
- '포스팅할 업체 선택'(F7) 목록을 생성된 포스트 보기(F8)와 똑같이 **📅 날짜 → 🕐 시간 → 지역·업종 → 업체** 4단으로 통일.
- 전체선택·2차필터·선택삭제·완료제거 등 모든 동작이 4단에서 정상 작동하도록 정리.

## v2.5.0 변경사항

### 예약발행 간격 오류 수정 (중요)
- 기존 예약을 모두 확인해서, 새 글이 어떤 예약과도 '설정 간격 − 20분'보다 가깝게 붙지 않도록 배치.
- 기존 예약이 촘촘하면 그 뒤로 간격대로 이어가고, 큰 빈 구간은 간격대로 채움 → 2시간 ±20분처럼 설정한 간격이 항상 지켜짐 (예전엔 50~70분 간격으로 끼어드는 문제).
- 예약 목록을 스크롤하며 전부 읽어옴 (가상 스크롤이라 일부만 읽히던 문제 보완).

### 생성된 포스트 보기 — 날짜 밑에 시간 단위 추가
- 목록을 📅 날짜 → 🕐 시간 → 지역·업종 → 업체 4단으로 세분화. 같은 시각 수집분이 한 묶음으로.

## v2.4.9 변경사항

### 지역 설정 — 구(區) 복구
- 앞서 제거했던 구를 다시 살림. 각 시 밑에 그 시의 구들이 묶여 나옴 (수원시 → 장안구·권선구·팔달구·영통구 등).

### 생성된 포스트 보기 — '포스트 글쓰기'와 동일한 3단 목록
- 목록을 F7 '포스팅할 업체 선택'과 똑같이 **📅 날짜 → 지역·업종 그룹 → 업체** 3단으로 통일 (별도 '날짜별' 탭 제거).
- 같은 날 수집분이 한 날짜 아래로 묶임. 포스트별 사용 API(딥시크/챗GPT/제미나이) 컬럼 표시.

## v2.4.8 변경사항

### 생성된 포스트 보기 — 날짜·시간 배치 + API 표시
- '날짜별' 탭을 생성 배치(날짜+시간) 단위로 묶음 — 같은 시각에 생성한 글이 한 묶음으로.
- 각 배치에 사용한 글쓰기 API(딥시크/챗GPT/제미나이) 표시 + 포스트별 'API' 컬럼 추가.
- (기존 글은 생성시각·API 정보가 없어 '날짜 미상'으로 표시되며, 이후 생성분부터 정상 기록.)

### 키워드마스터 연동 제거
- 시작 시 키워드마스터 핸드오프 파일을 읽어오던 수신 기능 제거 (양방향 연동 종료).

## v2.4.7 변경사항

### 지역 설정 간소화
- 전국 모든 '구(區)' 제거 — 시 단위로 통합 (예: 수원시 장안구·팔달구 → 수원시).
- 광역시(서울·부산·대구·인천·광주·대전·울산)도 구 없이 시 단위로만.

### 제목 순서 고정
- 제목이 항상 '주요키워드(지역+업종) → 업체명 → 수식어' 순서로 나오도록 고정 (수식어가 업체명보다 앞서던 문제 해결).

### 생성된 포스트 보기 — 날짜별 탭
- 생성된 포스트를 생성일(날짜)별로 묶어 보는 '날짜별' 탭 추가.

### 로그인 안정화
- 지역 크롤링 로그인이 옛 버튼 셀렉터(log.login)를 기다리다 실패하던 문제 수정 (신 loginBtn_row/column 대응). 세션 없는 2명의 아이디가 로그인창에서 멈추던 증상 해결.

## v2.4.6 변경사항

### 신규: 전체 자동 관리
- 댓글·이웃 관리 창에 '🔁 전체 자동 관리' 버튼 추가.
- 결제된 모든 아이디를 순서대로 자동 로그인하며 아이디마다 ① 서이추 자동 수락 ② 삭제요청 의심 댓글의 원글 자동 삭제 ③ 공감·자동답글을 일괄 처리.
- 미결제/만료 아이디는 자동 건너뜀, '중단' 버튼으로 언제든 정지. 자동 삭제 전 1회 확인창.

### 자동답글 문구 확장
- 자동답글 문구 풀 500 → 2000개로 확장 (봇 느낌 최소화, 중복 없이 다양화).

## v2.4.5 변경사항

### 예약발행 밀림 수정
- 예약 기준 시각(base_time)을 기존 예약 맨 뒤로 밀지 않고 '지금'부터 유지.
- 이미 예약된 10분 슬롯과 이번 배치 슬롯만 건너뛰어 빈 시간에 채움.
- 멀리 박힌 예약 하나 때문에 전체 일정이 며칠~몇 주씩 밀리던 문제 해결.
