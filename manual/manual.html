1장. 프로그램 소개

블로그마스터는 네이버 플레이스 기반 블로그 자동 포스팅 프로그램입니다. 지역과 업종 키워드만 입력하면 네이버 지도에서 업체 정보를 자동 수집하고, AI가 블로그 본문과 제목을 생성하며, 이미지까지 자동으로 다운로드한 뒤 네이버 블로그에 글을 작성·발행해 줍니다.

주요 기능

  • 네이버 지도 크롤링: 지역·업종 키워드로 업체 정보 (이름·주소·카테고리·근처역) 자동 수집
  • 전국 17개 시·도 자동 확장 — 시·도만 입력해도 전 구·시·군으로 확장 크롤
  • AI 본문 생성: GPT (OpenAI) 기반, 업종별 맞춤 프롬프트
  • 이미지 자동 삽입: 네이버 실사 사진 + Pixabay 보충 (중복 방지)
  • 장소 카드 자동 삽입: 업체명으로 네이버 장소 검색 → 본문에 자동 첨부
  • 예약 발행 자동 분기: 기존 예약 있으면 이어쓰기, 없으면 첫 글 즉시 + 나머지 예약
  • 다중 계정: 최대 9개 네이버 블로그 계정 관리
  • 다중 사용자: 한 PC에서 여러 사용자가 각자 계정·키 분리

지원 환경

항목요구사항
운영체제Windows 10 / 11 (64bit)
RAM최소 4GB (8GB 이상 권장)
Chrome 브라우저최신 버전 필수
인터넷안정적인 유선 권장
API 키OpenAI GPT API (유료), Pixabay (선택)

2장. 설치 & 첫 실행

1) 설치 파일 다운로드

  1. https://ezon.kr 접속 → 블로그마스터 카드의 "⬇ 다운로드" 클릭
  2. 또는 GitHub Releases 페이지에서 최신 버전의 BlogMaster_Install.exe 직접 다운로드
[스크린샷 자리 #01 — ezon.kr 홈페이지 다운로드 버튼]

2) 설치

  1. 다운로드한 BlogMaster_Install.exe 실행
  2. "바탕화면 아이콘 생성" 체크 후 "다음" — 권장
  3. 설치 완료 후 자동 실행
관리자 권한 없이 설치 가능합니다. 사용자 폴더(C:\Program Files\BlogMaster) 또는 현재 사용자만의 위치에 설치됩니다.

3) 첫 실행 시 알아둘 점

  • 실행하면 로그인 다이얼로그가 먼저 뜹니다
  • 처음에는 회원가입이 필요합니다 (관리자 사전 등록 또는 직접 가입)
  • 데이터(설정·로그·이미지)는 %APPDATA%\NaverBlogAuto에 저장됩니다
  • 크래시 로그는 %LOCALAPPDATA%\BlogMaster\crash_log.txt

3장. 로그인 & 회원가입

[스크린샷 자리 #02 — 로그인 다이얼로그]

로그인

  1. 아이디 입력
  2. 비밀번호 입력 (옆 👁 클릭하면 입력값 확인 가능)
  3. "로그인" 클릭
같은 아이디로 다른 PC에서 동시 접속할 수 없습니다. 다른 기기 접속이 감지되면 로그인이 거부됩니다. 10분간 미접속 시 세션이 자동 해제됩니다.

회원가입

  1. 로그인 다이얼로그 하단 "회원가입" 클릭
  2. 아이디 · 비밀번호 · 이름 · 생년월일 · 전화번호 · 이메일 입력
  3. 추천인 (선택 사항)
  4. 가입 완료 후 관리자 승인을 기다리거나 즉시 로그인

비밀번호 찾기

로그인 화면에서 "비밀번호 찾기" → 가입 시 입력한 이름 · 전화번호 · 이메일이 일치하면 등록된 이메일로 임시 비밀번호 발송.

4장. 환경설정 (API 키 & 계정)

메인 화면 우측 상단 "⚙ 설정" 버튼 또는 단축키로 환경설정 다이얼로그를 엽니다.

[스크린샷 자리 #03 — 환경설정 전체 화면]

4.1 AI 설정

이번 버전은 GPT (OpenAI) 전용입니다. Gemini는 안정성·가격 이슈로 비활성화되어 있습니다.

① GPT API 키 입력

  1. "GPT API 키" 칸에 sk-proj-...로 시작하는 키 붙여넣기
  2. 옆 👁 클릭 시 입력값 확인 가능
  3. 오른쪽 "키 발급받기 (유료)" 클릭 → OpenAI 페이지로 이동

② Pixabay API 키 입력 (선택)

  • Pixabay 키가 있으면 블로그 본문에 자동으로 관련 이미지가 삽입됩니다
  • 없으면 네이버 실사 사진만 사용 (1장)
  • https://pixabay.com/api/docs/ 에서 무료 발급 가능
관리자가 부여한 키가 있으면 본인 키 칸에 "●●●● 관리자 부여됨 ●●●●"로 표시됩니다. 본인이 직접 키를 발급받은 경우 이 칸에 본인 키를 입력하면 본인 키가 우선 사용됩니다.

4.2 계정 설정 (네이버 블로그)

최대 9개의 네이버 블로그 계정을 등록할 수 있습니다. 일반 사용자는 가입 등급에 따라 3·6·9개로 제한됩니다.

[스크린샷 자리 #04 — 환경설정 계정 탭]

각 슬롯에 입력

항목설명
블로그 IDblog.naver.com/여기 부분 (URL의 ID)
네이버 ID로그인용 아이디 (한 번 입력하면 변경 불가)
네이버 비밀번호로그인 비밀번호 (👁로 확인 가능)
블로그 카테고리발행할 카테고리 이름 (예: 추천, 게시판)
네이버 ID는 한 번 입력 후 변경할 수 없습니다 (관리자만 초기화 가능). 다른 사용자가 등록한 네이버 ID는 사용할 수 없습니다 (관리자 등록 ID는 예외).

4.3 비밀번호 변경

  1. 현재 비밀번호 입력
  2. 새 비밀번호 + 새 비밀번호 확인
  3. "비밀번호 변경" 클릭

각 비밀번호 칸 옆 👁로 입력값 확인 가능합니다.

4.4 (관리자) 이메일 발송 설정

관리자만 보이는 설정. 비밀번호 찾기 기능 사용을 위해 Gmail 앱 비밀번호로 발송 계정 등록.

5장. 메인 화면 구성

[스크린샷 자리 #05 — 메인 창 전체]

5.1 상단 헤더

  • 잔여기간: 계정 만료일까지 남은 일수 표시
  • 설정 / 로그아웃: 환경설정 다이얼로그 / 로그아웃
  • (관리자) 관리자 메뉴: 사용자 관리 다이얼로그 열기

5.2 좌측 패널 — 컨트롤

① 계정 콤보박스

등록된 네이버 블로그 계정 중 작업할 계정 선택. 슬롯 번호 + 블로그 ID 표시.

② 키워드 입력

크롤링할 키워드. 예: 서울 동물병원, 강남구 카페, 경기 헬스장

시·도만 입력 (예: 서울 동물병원)하면 25개 구로 자동 확장됩니다. 지역설정 다이얼로그에서 원하는 시·도·구만 선택 가능.

③ 지역설정 버튼 (▼)

17개 시·도 → 261개 구·시·군 트리에서 원하는 지역만 체크. 시·도 부모를 체크하면 하위 전체가 자동으로 토글됨.

④ 수집 갯수 / 제외 키워드

  • 수집 갯수: 한 키워드당 수집할 업체 수 (기본 100개)
  • 제외 키워드: 업체명/카테고리에 이 키워드 들어있으면 제외 (콤마 구분)

⑤ 발행 간격

  • 시간 + 분으로 예약 발행 간격 설정 (예: 2시간 30분)
  • "랜덤" 체크 시 ±10분 무작위 분산

5.3 우측 패널 — 실행 로그

[스크린샷 자리 #06 — 실행 로그 헤더]

헤더 영역

  • ● 상태 표시등: 현재 작업 상태 색 표시
  • 실행 로그: 패널 제목
  • 봇 ○1 ○2 ●3: 크롤링 봇 개수 (기본 3 / 1·2·3 선택, 자동 저장)
  • 크롤링 N개 | 포스트생성 N개 | 발행 N개: 누적 카운터

본문 영역

  • 크롤링 현황: 봇별 작업 상태 카드 + 봇별 로그
  • 포스트 현황: 생성/발행 진행 상황
  • 전체 로그: 시간별 모든 동작 로그

6장. 키워드 크롤링 (F5)

6.1 기본 흐름

  1. 좌측 패널의 키워드 입력 칸에 키워드 입력 (예: 서울 동물병원)
  2. 수집 갯수 조정 (기본 100개)
  3. 지역설정 (▼)에서 원하는 시·도/구 선택 (선택사항)
  4. 크롤 시작 (F5) 클릭
  5. 봇이 자동으로 분배되어 병렬 크롤 시작

6.2 시·도 자동 확장

키워드가 (시·도) (업종) 형태면 자동으로 모든 구/시·군으로 확장됩니다.

입력실제 크롤되는 키워드
서울 동물병원서울 강남구 동물병원, 서울 강동구 동물병원 ... (25개 구)
경기 헬스장경기 수원시 헬스장, 경기 성남시 헬스장 ... (48개 시·군)
강남구 카페강남구 카페 (확장 없음)

6.3 지역설정 다이얼로그

[스크린샷 자리 #07 — 지역설정 다이얼로그]
  • 17개 시·도 → 각 구·시·군 트리
  • 처음 열면 시·도가 모두 접힌 상태 — 클릭해서 펼치면 하위 구·시·군이 나타남 (v1.6.8 개선)
  • 시·도 부모 체크 → 하위 전체 토글
  • "전체 선택 / 전체 해제" 버튼
  • 선택 후 "확인" → 다음 크롤부터 적용 (자동 저장)
과거에는 17개 시·도가 다 펼쳐져 있어 스크롤이 길었습니다. v1.6.8부터는 원하는 시·도만 클릭해서 펼치는 방식으로 변경되어 훨씬 빠르게 선택할 수 있습니다.

6.4 병렬 크롤 (봇)

실행로그 헤더의 봇 1/2/3 라디오로 동시 작업 수 선택. 키워드 개수와 봇 개수 중 작은 값이 적용됩니다.

RAM 4GB 미만 PC는 자동으로 봇 1개로 제한됩니다. 4~8GB는 봇 2개, 8GB 이상은 사용자 설정 그대로(최대 3개). 로그에 "RAM N.NGB 감지 — 봇 N개 → N개로 자동 제한" 메시지가 표시됩니다.

6.5 제외 키워드

업체명 또는 카테고리에 다음 키워드가 들어있으면 수집에서 제외됩니다.

  • 콤마로 여러 개 구분: 가맹점, 광고, 임시휴업
  • 대소문자 무시
  • 부분 일치 (예: "광고" → "광고 대행" 포함)

6.6 이어 크롤 (재개)

같은 키워드로 다시 F5 누르면 이전에 수집한 업체는 건너뛰고 이어서 수집합니다. 완전히 새로 시작하려면 결과 보기에서 데이터를 삭제한 뒤 재시작하세요.

6.7 크롤 결과 보기

"결과 보기" 버튼 클릭 → 엑셀 표 형태 다이얼로그 → "엑셀로 복사" 클릭 시 클립보드에 복사되어 Excel에 붙여넣기 가능.

네이버에서 짧은 시간에 너무 많은 크롤을 시도하면 IP가 일시적으로 차단될 수 있습니다. 하루 5,000개 이상 수집 시 적절한 휴식을 권장합니다.

7장. 포스트 생성 (F7)

7.1 기본 흐름

  1. 크롤이 완료된 상태에서 포스트 글쓰기 (F7) 클릭
  2. 업체 선택 다이얼로그 — 지역기반 / 키워드기반 탭에서 원하는 업체 체크
  3. "선택한 N개로 글쓰기 시작" 클릭
  4. AI가 업체별로 본문·제목 생성 시작 (병렬 5개 워커)
  5. 1개 완료될 때마다 자동 저장 — F8에서 실시간 확인 가능

7.2 업체 선택 다이얼로그

[스크린샷 자리 #08 — F7 업체 선택]
  • 지역기반 탭: 동별로 그룹화 (강남구 논현동 등)
  • 키워드기반 탭: 키워드별로 그룹화 (서울 강남구 동물병원 등)
  • 부모(그룹) 체크 → 하위 전체 자동 체크
  • "전체 선택 / 전체 해제" 버튼으로 일괄 토글
  • 제목 형식 선택: 동 / 역 / 구 / 자유입력

7.2.1 날짜별 일괄 선택 (v1.6.8 신규)

상단에 수집 날짜별 체크박스가 자동으로 생성됩니다.

[스크린샷 자리 #08-A — 날짜별 체크박스 패널]
  • 최근 수집한 날짜부터 차례로 표시 (예: 06-10 (47), 06-09 (30))
  • 괄호 안 숫자는 그날 수집된 업체 수
  • 날짜 체크박스 클릭 → 그날 수집한 업체 전부 자동 체크/해제
  • 여러 날짜를 동시에 선택 가능
매일 크롤링하시는 분들에게 매우 유용합니다. 어제 수집한 50개를 일일이 체크할 필요 없이 06-10 한 번 클릭으로 끝.

7.2.2 2차 필터 (v1.6.8 신규)

상단의 주황색 "2차 필터" 버튼 클릭 — 카테고리 기준으로 무관한 업체를 일괄 제거합니다.

[스크린샷 자리 #08-B — 2차 필터 다이얼로그]

왜 필요한가?

네이버 지도는 키워드 단어만 매칭되어도 검색결과에 포함시킵니다. 예를 들어 "포장이사"로 검색하면 진짜 포장이사 업체 외에 다음과 같은 무관 업체도 잡힙니다:

  • 배관/냉난방공사 (업체 설명에 "포장이사"가 우연히 들어감)
  • 전문건설업
  • 컴퓨터/모니터 수리
  • "톡톡", "쿠폰" 같은 부가 정보가 카테고리로 잡힌 경우

사용 방법

  1. "2차 필터" (주황색) 버튼 클릭
  2. 현재 표시된 모든 업체의 카테고리 + 개수가 목록으로 나타남
  3. 카테고리는 개수가 많은 순서로 정렬됨 (포장이사 10개 → 용달 4개 → ...)
  4. 원치 않는 카테고리(예: 배관, 컴퓨터)의 체크 해제
  5. "적용" 클릭 → 해제된 카테고리의 모든 업체가 자동으로 체크 해제됨
"전체 선택" 한 뒤 "2차 필터"로 무관 카테고리만 해제하면 빠르게 깨끗한 목록을 만들 수 있습니다.
2차 필터는 이미 선택된 항목을 카테고리 단위로 정리하는 도구입니다. 새로 추가하는 것이 아니라 빼는 용도입니다.

7.4 본문 생성 과정

  1. 업종에 맞는 프롬프트 자동 선택 (예: 동물병원 → 치과 프롬프트 기반)
  2. 업체정보를 프롬프트 변수에 채워 GPT 호출
  3. 본문 (1,500자 이상) + 제목 후보 10개 생성
  4. 제목 후보 중 블랙리스트(지역·업종·"내돈내산" 등) 단어 없는 것 랜덤 선택
  5. 본문에 [이미지] 마커 자동 삽입 (3~5개)
  6. 네이버 실사 사진 1장 + Pixabay 이미지 보충 다운로드
  7. 생성된 포스트를 디스크에 즉시 저장

7.5 이미지 자동 선택 로직

  • 1순위: 네이버 플레이스 실사 사진 1장 (place_id 있을 때)
  • 2순위: Pixabay 검색 (업종별 영문 키워드 매핑)
  • 이미지 ID 중복 방지 — 같은 사진 두 번 안 받음 (계정별 별도 추적)
  • 키워드 매핑 실패 시 GPT로 한글→영어 번역 후 재검색
  • 매핑·번역 모두 실패 시 이미지 없이 진행 (엉뚱한 사진 차단)

7.6 글쓰기 중단

"글쓰기 중단" 버튼 또는 F6으로 즉시 중단. 이미 생성된 포스트는 보존됩니다.

8장. 포스트 보기 & 자동 발행 (F8)

8.1 다이얼로그 구성

[스크린샷 자리 #09 — F8 생성된 포스트 보기]
  • 상단 정보: ● 초록=생성됨, [완] 파랑=발행완료, ● 빨강=미생성
  • 지역기반 / 키워드기반 탭: 그룹별 트리 뷰
  • 부모 체크 → 하위 전체 토글
  • 업체 더블클릭 → 단일 포스트 미리보기 (제목·본문·이미지 편집)

8.2 단일 포스트 미리보기

업체를 더블클릭하면 단일 포스트 편집 다이얼로그가 뜹니다.

[스크린샷 자리 #10 — 단일 포스트 편집]
  • 제목 편집: 상단 입력칸에서 직접 수정
  • 본문 편집: 큰 텍스트 영역에서 수정
  • 이미지 패널: 썸네일 클릭 → 큰 화면, "이미지 변경" → Pixabay 재검색

8.3 자동 포스팅 시작

  1. 포스팅할 업체 체크
  2. (선택) 우측 상단의 "랜덤포스팅" 토글 — 발행 순서 무작위화
  3. 하단 "자동 포스팅" 버튼 클릭
  4. 예약 발행이 이미 있는지 자동 확인
  5. 없으면 첫 글은 즉시 발행 + 나머지는 설정 간격으로 예약
  6. 있으면 그 뒤로 이어서 예약

8.4 발행 진행 상황

  • 크롬 창이 자동으로 뜨고 네이버 로그인 → 글쓰기 → 발행/예약
  • 실행 로그 우측 패널의 "포스트 현황"에서 실시간 확인
  • 각 글당 약 30~50초 소요 (속도 개선 적용)
  • 발행 완료 시 F8에서 [완] 표시로 확인 가능

8.5 발행 중단

"포스팅 중단" 버튼 또는 F6 → 1초 이내 즉시 중단. 활성 크롬 창도 강제 종료됩니다.

8.6 완료 항목 관리

  • 선택 삭제: 체크된 업체 + 그 포스트를 완전 삭제
  • 완료 목록 삭제: [완] 표시된 발행 완료 포스트만 일괄 삭제 (★새 기능)
  • 전체 삭제: 현재 표시된 모든 업체 + 포스트 삭제
발행 완료된 포스트는 자동 포스팅 시 자동으로 제외됩니다 (재발행 방지). 사용자에게 "이미 발행 완료 N개 제외됨" 안내가 표시됩니다.

8.7 네이버 예약 발행 100개 한도

네이버는 계정당 최대 100개까지만 예약 발행을 허용합니다. 프로그램이 99개 도달 시 자동으로 포스팅을 중단합니다.

9장. 예약발행 자동 분기

v1.6.6부터 예약 발행 옵션이 단순화되었습니다. 사용자가 별도 체크할 항목 없이 자동으로 분기합니다.

분기 로직

기존 예약 상태동작

10장. 봇 개수 설정 & 저사양 PC 대응

10.1 봇 개수 선택

10.2 RAM 자동 캡

저사양 PC 보호를 위해 RAM에 따라 봇 개수가 자동 제한됩니다.

시스템 RAM최대 봇 개수
4GB 미만1개
4~8GB2개
8GB 이상3개 (사용자 설정)
Chrome 인스턴스 하나당 약 300~500MB의 RAM을 사용하므로 저사양 PC에서 봇 3개를 띄우면 시스템이 멈출 수 있어 자동 제한합니다. 제한 적용 시 로그에 안내 메시지가 표시됩니다.

10.3 Chrome 메모리 절약 옵션

자동 적용되는 옵션:

  • --mute-audio: 오디오 처리 비활성화
  • --disable-background-networking: 백그라운드 네트워크 사용 차단
  • --disable-default-apps: 기본 앱 비활성화
  • --disable-sync: 동기화 비활성화
  • --disable-extensions: 확장 프로그램 비활성화
